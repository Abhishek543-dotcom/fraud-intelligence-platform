## Context

The project is a Fraud Intelligence Platform with:
- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS + Lucide icons + @tanstack/react-query
- **Backend**: FastAPI (Python) with routers under `/api`
- **Data layer**: Apache Iceberg tables managed via Nessie catalog + Spark + MinIO
- **Existing table listing**: `GET /api/metrics/nessie/tables` already queries Nessie REST API and returns table names/types/namespaces
- **No SQL execution endpoint exists** — the `IcebergQueryService` is a stub

The frontend has 6 tabs in the sidebar (`Layout.tsx`). Adding a "SQL Editor" tab requires both backend (SQL execution endpoint) and frontend (new component) work.

For SQL execution, the best approach is to use **PyIceberg** (lightweight Python library) to query tables directly via the Nessie catalog + MinIO storage — no Spark Thrift Server needed. This keeps the backend self-contained without requiring Spark connectivity for simple queries.

## Implementation Steps

### 1. Add `pyiceberg` dependency to backend
**File**: `backend/requirements.txt`
- Add `pyiceberg[s3]==0.7.1` (provides direct table reads from S3/MinIO via Nessie catalog)

### 2. Create SQL query execution router
**File**: `backend/app/routers/sql_editor.py` (new)
- `GET /api/sql/tables` — list all Iceberg tables from Nessie (reuse `_nessie_state` logic but with schema details)
- `POST /api/sql/execute` — accept SQL string, execute against Iceberg tables using PyIceberg + DuckDB (PyIceberg uses DuckDB for SQL queries on Arrow tables), return columnar results
- Include query timeout (30s max), row limit (1000 rows default), and basic SQL sanitization (read-only: only SELECT/SHOW/DESCRIBE allowed)
- `GET /api/sql/tables/{namespace}/{table}/schema` — return column names/types for a specific table

### 3. Register the new router in the FastAPI app
**File**: `backend/app/main.py`
- Import and include the `sql_editor` router with prefix `/api`

### 4. Create the frontend SQL Editor component
**File**: `frontend/src/components/SqlEditor.tsx` (new)
- **Left panel (sidebar)**: Tree view of Iceberg tables grouped by namespace (bronze/silver/gold), expandable to show columns with data types
- **Main area (top)**: SQL textarea with monospace font, syntax highlighting placeholder, and "Run" button
- **Main area (bottom)**: Results table showing query output with column headers and scrollable rows
- Include loading state, error display, empty state, and row count indicator
- Style using Tailwind CSS matching existing dark theme (`bg-gray-900`, `border-gray-800`, etc.)

### 5. Add API service functions for SQL editor
**File**: `frontend/src/services/api.ts`
- `fetchIcebergTables()` — GET `/api/sql/tables`
- `fetchTableSchema(namespace, table)` — GET `/api/sql/tables/{namespace}/{table}/schema`
- `executeQuery(sql)` — POST `/api/sql/execute`

### 6. Register route and nav item
**File**: `frontend/src/App.tsx`
- Import `SqlEditor` component
- Add `<Route path="/sql" element={<SqlEditor />} />`

**File**: `frontend/src/components/Layout.tsx`
- Add `{ to: '/sql', label: 'SQL Editor', icon: Database }` to `navItems` array (using `Database` icon from lucide-react)
