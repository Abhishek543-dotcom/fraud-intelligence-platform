## Context

Investigated the platform end-to-end. Key findings:

**Frontend**
- `Layout.tsx` is a left sidebar (not top tabs). Likely render bug = overflow / icon‑label cropping. Need responsive polish.
- `MetricsPanel`, `KafkaThroughput`, backend `/api/metrics/*` all return **random** values (`Math.random()` / `random.randint`). That's why "Grafana doesn't work and frontend looks fake" — they just aren't wired to real sources.

**Why MinIO/Nessie/Spark are empty**
- `minio-init` script uses `docker exec fraud-minio mc …` *from inside* the init container (no docker socket → fails) → buckets never created → Spark can't write checkpoints/data.
- `fraud-spark-master` and `fraud-spark-worker` have **zero volume mounts** (verified via `docker inspect`) — the streaming/lakehouse Python code is invisible to Spark. spark-submit cannot find the job file.
- Airflow DAG `fraud_pipeline_dag.py` BashOperator runs `spark-submit /opt/spark-jobs/streaming_pipeline.py` — that path doesn't exist; real path is `/opt/airflow/streaming/spark-jobs/fraud_detection_streaming.py`. Airflow container also can't reach `spark-master:7077` without a Spark connection.
- The previous "simulator" container we shut off → no producer at all → Kafka topic `transactions_raw` is empty (verified). The real producer code lives at `streaming/producers/transaction_simulator.py` (uses relative imports → needs to run as `python -m streaming.producers.transaction_simulator`).

### Phase 2 — Backend: real metrics endpoints (`backend/app/routers/metrics.py`)
Replace random‑data endpoints with real probes:
- `GET /api/metrics/kafka/topics` — uses `KafkaAdminClient` + `KafkaConsumer` end_offsets to return per‑topic partition count, total messages, lag.
- `GET /api/metrics/kafka/throughput-history` — backend background task samples Kafka topic offsets every 5s, keeps a 5‑min ring buffer; endpoint returns the series.
- `GET /api/metrics/spark/master` — proxies `http://spark-master:8080/json/` (workers, active/completed apps, cores, mem).
- `GET /api/metrics/minio/buckets` — `boto3` `list_buckets()` + `list_objects_v2` summing `Size` per bucket.
- `GET /api/metrics/nessie/tables` — `httpx` against `http://nessie:19120/api/v2/trees/main/entries` (recursive across namespaces).
- `GET /api/metrics/ml/health` — proxies `http://ml-service:8000/health` and parses `/metrics` Prometheus text for `fraud_predictions_total`, latency histogram.
- Rewrite `GET /api/metrics/overview` to compute totals from the above sources (with safe fallback to 0).

Add `boto3`, `httpx` to `backend/requirements.txt` if missing.
